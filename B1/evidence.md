
# Evidence Summary

The evidence collected for this activity demonstrates five common weak security implementations found in real-world systems and applications.

**Default Credentials:**
A screenshot from official NETGEAR documentation was collected showing default usernames and passwords such as "admin" and "password". This demonstrates how attackers can gain unauthorized access when default credentials are not changed after installation.

**HTTP Instead of HTTPS:**
A screenshot from the website neverssl.com was collected showing a browser warning that the connection was not secure. This demonstrates the risks associated with transmitting information over unencrypted HTTP connections.

**Weak Password Policies:**
Evidence was collected demonstrating the use of weak passwords that are commonly accepted by systems. Weak passwords increase the likelihood of successful brute-force and dictionary attacks.

**Hardcoded Secrets in Source Code:**
A screenshot of source code containing embedded credentials was collected. This demonstrates how API keys, passwords, and authentication tokens can be exposed when secrets are stored directly within source code.

**Outdated and Unpatched Software:**
A screenshot from Microsoft's Windows 8.1 end-of-support documentation was collected. The documentation confirms that security updates are no longer provided, demonstrating the risks associated with running unsupported software.

Together, these examples highlight how poor security practices can significantly increase the attack surface of systems and organizations. Many cybersecurity incidents can be prevented by implementing basic security best practices and maintaining secure configurations.
