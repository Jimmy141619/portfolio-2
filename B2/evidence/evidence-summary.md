# B2 – Evidence Summary

This activity investigated five strong security implementations that are commonly used to improve cybersecurity and reduce the risk of cyberattacks.

## Multi-Factor Authentication (MFA)

I reviewed Multi-Factor Authentication (MFA), which requires users to provide an additional verification factor beyond a password when accessing an account. MFA significantly reduces the risk of unauthorized access resulting from stolen or compromised passwords.

---

## Password Managers

I reviewed Bitwarden's password manager documentation, which demonstrates how password managers generate and securely store strong, unique passwords. Password managers reduce password reuse and help protect users against credential theft and account compromise.

---

## HTTPS/TLS Encryption

I reviewed a website using HTTPS and TLS encryption, indicated by the secure connection icon within the browser. HTTPS protects sensitive information by encrypting communications between users and websites, preventing unauthorized interception of data.

---

## Role-Based Access Control (RBAC)

I reviewed Linux user and permission management, which implements access control through users, groups, and permission assignments. This security mechanism ensures users can only access resources necessary for their role and responsibilities.

---

## Automatic Security Updates

I reviewed operating system update functionality, which automatically installs security patches and software updates. Automatic updates reduce the likelihood of exploitation by ensuring known vulnerabilities are patched promptly.

---

## Reflection

This activity demonstrated that effective cybersecurity relies on implementing multiple layers of security controls. Multi-Factor Authentication, password managers, HTTPS encryption, access control mechanisms, and automatic updates all contribute to reducing cyber risk. Together, these security implementations form a strong defense against common threats and attacks.
