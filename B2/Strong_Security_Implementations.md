# B2 – Discover 5 Unique Strong Security Implementations

## Introduction

Strong security implementations help organizations protect systems, data, and users from cyber threats. This activity investigates five commonly used security implementations that improve the confidentiality, integrity, and availability of information systems.

---

# 1. Multi-Factor Authentication (MFA)

## Description

Multi-Factor Authentication requires users to provide two or more forms of verification before accessing an account or system.

## Security Benefits

* Reduces the risk of unauthorized access
* Protects against stolen passwords
* Provides an additional layer of security


# 2. Password Managers

## Description

Password managers securely store and generate strong passwords for user accounts.

## Security Benefits

* Prevents password reuse
* Encourages strong unique passwords
* Reduces credential theft risks



# 3. HTTPS/TLS Encryption

## Description

HTTPS uses Transport Layer Security (TLS) to encrypt communications between users and websites.

## Security Benefits

* Protects sensitive information during transmission
* Prevents eavesdropping attacks
* Ensures confidentiality and integrity of data


# 4. Role-Based Access Control (RBAC)

## Description

Role-Based Access Control restricts access to resources based on a user's assigned role and permissions.

## Security Benefits

* Enforces least privilege
* Reduces insider threats
* Improves access management


# 5. Automatic Security Updates

## Description

Automatic updates ensure software and operating systems receive security patches as soon as they become available.

## Security Benefits

* Reduces exposure to vulnerabilities
* Improves system security
* Protects against known exploits
