# B3 – Discover 3 Proactive Security Implementations

## Introduction

Proactive security implementations are security measures designed to identify, prevent, or mitigate cyber threats before they result in successful attacks. Unlike reactive controls, proactive security mechanisms continuously monitor systems, identify risks, and reduce potential vulnerabilities.

---

# 1. Intrusion Detection Systems (IDS)

## Description

An Intrusion Detection System (IDS) monitors network traffic and system activity to identify suspicious behavior and potential security threats.

## Security Benefits

* Detects malicious activity
* Identifies unauthorized access attempts
* Provides early warning of attacks

## Evidence

Screenshot of Snort, Suricata, Zeek, or IDS documentation.

---

# 2. Threat Intelligence Platforms

## Description

Threat intelligence platforms collect, analyze, and distribute information about emerging cyber threats, attack techniques, and malicious actors.

## Security Benefits

* Improves threat awareness
* Helps organizations anticipate attacks
* Supports proactive defense strategies

## Evidence

Screenshot of a threat intelligence platform or threat intelligence report.

---

# 3. Automated Patch Management

## Description

Automated patch management systems automatically deploy security updates and software patches to reduce exposure to known vulnerabilities.

## Security Benefits

* Reduces vulnerability exposure
* Improves compliance
* Minimizes human error

## Evidence

Screenshot of patch management software or automatic update functionality.

---

# Reflection

This activity demonstrated the importance of proactive cybersecurity measures. By identifying threats early, monitoring for suspicious activity, and applying security updates automatically, organizations can reduce the likelihood and impact of cyberattacks.
